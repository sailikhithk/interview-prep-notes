LEETCODE 10 HARD

Regular Expression Matching

Given an input string s and a pattern p, implement regular expression matching with support for '.' (any single character) and '*' (zero or more of the preceding element). The match must cover the entire input string.

Dynamic Programming String Recursion / Memoization
TimeO(M·N)
SpaceO(M·N)

1. Problem Statement

Given an input string s and a pattern p, implement regular expression matching where:

  • '.' matches any single character.
  • '*' matches zero or more of the preceding element.

The matching should cover the entire input string (not partial). The '*' always has a valid character or '.' before it.

Examples: 
Input:  s = "aa",   p = "a"     → false   "a" does not match the whole "aa"
Input:  s = "aa",   p = "a*"    → true    '*' repeats 'a' once → "aa"
Input:  s = "ab",   p = ".*"    → true    ".*" = "zero or more of any char"
Input:  s = "aab",  p = "c*a*b"  → true   "c*" → 0 c, "a*" → 2 a, then "b"
Input:  s = "mississippi", p = "mis*is*p*."  → false
Constraints: 1 ≤ s.length ≤ 20 · 1 ≤ p.length ≤ 20 · s contains only lowercase letters · p contains lowercase letters, '.', and '*' · every '*' has a valid preceding element.
2. Pattern Recognition
Pattern 2D Dynamic Programming on Two Strings

When a decision at each character depends on a choice ("use the star, or skip it") and overlapping subproblems appear, build a grid over the two strings.

How to spot it:

Two sequences (here a string and a pattern) and you ask "does prefix of A match prefix of B?".

A token ('*') creates a branch: zero occurrences vs. one-or-more occurrences — classic "take it or leave it".

Naive recursion re-solves the same (i, j) suffix pairs → overlapping subproblems → memoize / tabulate.

Cousins: Edit Distance (72), Wildcard Matching (44), LCS (1143), Interleaving String (97).

3. Pattern Blueprint Template

Generic 5-step template for "match two strings" 2D DP:

1

Define state: dp[i][j] = does s[0..i) match p[0..j). Dimensions (m+1) × (n+1).

2

Base case: dp[0][0] = true (empty matches empty). Seed the first row for patterns like a*b*c* that can match empty.

3

Transition (no star): if p[j-1] == s[i-1] or p[j-1] == '.', inherit dp[i-1][j-1].

4

Transition (star at p[j-1]): zero copies → dp[i][j-2]; one+ copies → if p[j-2] matches s[i-1], OR in dp[i-1][j].

5

Answer: dp[m][n]. Fill the grid row-major; every cell is O(1).

4. How to Think About It
1

Start from the recursive question: "does s match p?" Peel one pattern token at a time, not one character.

2

The first-char match is firstMatch = i<m and (p[j]==s[i] or p[j]=='.'). This single boolean drives everything.

3

If the next pattern char is '*', you have two universes: skip the x* block entirely (j += 2), or consume one s char if firstMatch and stay on the same star.

4

Else (no star), require firstMatch and advance both by one. → Recursion has overlapping (i, j) states → flip it into a bottom-up grid.

5. Real-World Analogy

Think of the pattern as an airport security checklist and the string as a line of passengers. A normal pattern char is a checkpoint that admits exactly one passenger of a specific type. A '.' checkpoint admits any one passenger. The tricky one is the '*' checkpoint — it is a flexible lane that can wave through zero passengers (closed lane) or many passengers of the preceding type, one after another.

At each flexible lane you make a branching bet: "do I close this lane now and move to the next checkpoint, or do I send one more matching passenger through and keep the lane open?" The DP table is the master log recording, for every (passengers-processed, checkpoints-passed) pair, whether a valid arrangement exists — so you never re-investigate the same situation twice.

Mapping: passengers = s · checklist = p · flexible lane = x* · close the lane = dp[i][j-2] · send one more through = dp[i-1][j].

6. Build Your Intuition

Q: Why does '*' look back two columns (j-2) for the "zero" case?

Because x* is a two-character unit. "Zero copies of x" means deleting the whole x* block, so we jump back past both the char and the star → dp[i][j-2].

Q: Why does the "one or more" case look at dp[i-1][j] and not dp[i-1][j-2]?

Because the star can be reused. After consuming s[i-1] with the star, we still sit on the same star (column j) to maybe consume more. We only drop one string char → dp[i-1][j].

Q: Why seed the first row but not the first column?

An empty pattern can never match a non-empty string, so the first column (j=0, i>0) is all false. But a non-empty pattern like a* can match the empty string, so row 0 must be computed.

7. How It Works (Step by Step)

Tabulating s = "aa", p = "a*" (grid is 3×3):

1

dp[0][0]=T — empty matches empty.

2

dp[0][2]: pattern "a*" vs empty — star takes zero a's → dp[0][0]=T. So "a*" matches "".

3

dp[1][1]: "a" vs "a", chars equal → inherit dp[0][0]=T.

4

dp[1][2]: star — zero case dp[1][0]=F; one+ case p[0]='a'==s[0]='a' → dp[0][2]=T. Result T.

5

dp[2][2]: star one+ case reuses dp[1][2]=T — second a consumed by the same star. Answer = dp[2][2] = T.

10. Complexity Deep Dive

Time — O(M·N)

There are (m+1)(n+1) states and each cell is computed in O(1) (a constant number of lookups: dp[i][j-2], dp[i-1][j], dp[i-1][j-1]). No state is recomputed.

Space — O(M·N) → O(N)

The full grid is O(M·N). Because each row depends only on the current and previous row, you can roll to two rows for O(N) space. Memoized top-down recursion is also O(M·N) for the cache plus O(M+N) call stack.

Why naive recursion explodes

Without memoization, each '*' spawns a branch (skip vs consume), so the call tree can be exponential — roughly O((m+n) · 2^(m+n/2)) in adversarial cases like s="aaaaaaaaaaaaab", p="a*a*a*a*...". The DP table collapses all those repeats into M·N work.

11. From Brute Force to Optimal

Naive — Pure Recursion

Recurse on (i, j), branching on every star. Re-solves the same suffix pairs over and over.

Time: exponential · Space: O(M+N) stack

Optimal — Tabulated DP

Cache every (i, j) in a grid; each computed once. Same recurrence, no repeats.

Time: O(M·N) · Space: O(M·N) or O(N)

For 20×20 inputs that is ~441 cells instead of potentially millions of recursive calls — the speedup is the whole point of DP.

12. Edge Cases & Gotchas

Empty string, non-empty pattern

s="", p="a*b*" must return true. The first-row seeding handles this; forgetting it is the #1 bug.

Star as the first pattern char

The constraints forbid a leading '*' (every star has a preceding element). Still, guard j-2 ≥ 0 so you never index negatively.

'.' vs '.*'

'.' consumes exactly one char; '.*' consumes zero-or-more of any char. Treat '.' as a wildcard inside the match() helper, not as a literal.

Off-by-one in indices

dp[i][j] refers to s[i-1] and p[j-1] (1-indexed table over 0-indexed strings). Mixing the two is the second most common bug.

13. Flashcards Q&A
What is the DP state definition?

dp[i][j] = true iff the first i chars of s match the first j chars of p. Answer is dp[m][n].

Star transition in one line?

dp[i][j] = dp[i][j-2] or (match(i,j-1) and dp[i-1][j]).

Non-star transition?

If p[j-1]==s[i-1] or p[j-1]=='.' then dp[i][j]=dp[i-1][j-1], else false.

What does match(i,j) mean?

i>0 and (p[j-1]=='.' or p[j-1]==s[i-1]) — does pattern char j cover string char i.

Time & space?

O(M·N) time, O(M·N) space (reducible to O(N) with a rolling row).

14. Where It Is Used (Real World)

grep / ripgrep / awk

Line matching engines compile patterns into automata; the DP form is the textbook backtracking baseline.

Lexers & compilers

Tokenizers describe tokens as regexes; * is Kleene star, the same "zero or more" operator.

Log / SIEM filtering

Splunk, Datadog, and ELK route and alert on log lines using pattern matchers built on these ideas.

Bioinformatics

Sequence/motif matching against DNA uses the same prefix-DP family (alongside edit distance).

15. Common Mistakes

Forgetting to seed row 0 for patterns that match empty (a*, a*b*).

Treating '*' like wildcard-'*' (LeetCode 44) which matches any sequence by itself — here it modifies the preceding char.

Using dp[i-1][j-2] instead of dp[i-1][j] for the "consume one" case — you must stay on the same star.

Indexing p[j] / s[i] instead of p[j-1] / s[i-1] in the 1-indexed table.

Returning on the first full-string match in recursion without requiring the pattern to be exhausted too.

16. Interview Tips

Start with the top-down recursion first — it's easier to derive live. Then say "this has overlapping subproblems" and convert to a table.

Write the match(i,j) helper out loud first; it isolates the '.' logic and de-clutters the main recurrence.

Explicitly call out the two star branches ("skip the block" vs "consume one and stay") — interviewers want to hear that insight.

Hand-trace s="aa", p="a*" on the whiteboard to prove the empty-row seeding and star-reuse.

19. Where This Pattern Meets System Design

Regex engines as compiled automata

In high-throughput systems you don't re-run DP per line. You compile the pattern once into an NFA/DFA (Thompson construction, RE2) and stream input through it — turning O(M·N)-per-match into O(N)-per-line with a shared, cached automaton. The DP table here is the conceptual model the compiled engine optimizes away.

Log routing & alerting pipelines

Datadog/Splunk/ELK match millions of log lines/sec against pattern rules. They use RE2-style linear-time engines (no catastrophic backtracking) precisely because the naive recursive form is exponential — the same explosion Section 10 describes, now a production reliability concern.

ReDoS as a security failure mode

An attacker-supplied pattern with nested stars ((a*)*) against a backtracking engine is a denial-of-service vector. System design answer: bound execution with linear-time engines, timeouts, and input-size limits — a direct application of "know your worst-case complexity."

20. Code Solutions

Python — Bottom-up DP

class Solution:
    def isMatch(self, s: str, p: str) -> bool:
        m, n = len(s), len(p)
        dp = [[False] * (n + 1) for _ in range(m + 1)]
        dp[0][0] = True

        def match(i, j):  # does p[j-1] cover s[i-1]?
            return i > 0 and (p[j-1] == '.' or p[j-1] == s[i-1])

        for i in range(m + 1):
            for j in range(1, n + 1):
                if p[j-1] == '*':
                    dp[i][j] = dp[i][j-2]                 # zero copies
                    if match(i, j-1):                     # one or more
                        dp[i][j] = dp[i][j] or dp[i-1][j]
                elif match(i, j):
                    dp[i][j] = dp[i-1][j-1]
        return dp[m][n]

Java

class Solution {
    public boolean isMatch(String s, String p) {
        int m = s.length(), n = p.length();
        boolean[][] dp = new boolean[m + 1][n + 1];
        dp[0][0] = true;
        for (int i = 0; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (p.charAt(j - 1) == '*') {
                    dp[i][j] = dp[i][j - 2];                       // zero copies
                    if (match(s, p, i, j - 1))                     // one or more
                        dp[i][j] = dp[i][j] || dp[i - 1][j];
                } else if (match(s, p, i, j)) {
                    dp[i][j] = dp[i - 1][j - 1];
                }
            }
        }
        return dp[m][n];
    }
    private boolean match(String s, String p, int i, int j) {
        return i > 0 && (p.charAt(j - 1) == '.' || p.charAt(j - 1) == s.charAt(i - 1));
    }
}

class Solution {
public:
    bool isMatch(string s, string p) {
        int m = s.size(), n = p.size();
        vector<vector<bool>> dp(m + 1, vector<bool>(n + 1, false));
        dp[0][0] = true;
        auto match = [&](int i, int j) {

                if (p[j - 1] == '*') {

                } else if (match(i, j)) {
                    dp[i][j] = dp[i - 1][j - 1];
                }
            }
        return dp[m][n];

};

TypeScript — Top-down memoized

function isMatch(s: string, p: string): boolean {
  const m = s.length, n = p.length;
  const memo = new Map<string, boolean>();
  const dp = (i: number, j: number): boolean => {
    if (j === n) return i === m;
    const key = i + ',' + j;
    if (memo.has(key)) return memo.get(key)!;
    const first = i < m && (p[j] === s[i] || p[j] === '.');
    let ans: boolean;
    if (j + 1 < n && p[j + 1] === '*') {
      ans = dp(i, j + 2) || (first && dp(i + 1, j));   // skip block | consume one
    } else {
      ans = first && dp(i + 1, j + 1);
    }
    memo.set(key, ans);
    return ans;
  };
  return dp(0, 0);
}
21. Follow-Up Questions
Reduce space to O(N)?

Each row depends only on the current row (for j-2) and the previous row (for i-1). Keep two 1-D arrays prev and cur and swap them each iteration → O(N) space.

How does this differ from Wildcard Matching (44)?

In LC44, '*' stands alone and matches any sequence; '?' matches one char. Here '*' modifies the preceding token. The grids look similar but the star transition differs.

Support '+' (one or more) and '?' (optional)?

Compile x+ to xx* and x? to a zero-or-one branch (dp[i][j-1] or matched dp[i-1][j-1]), or generalize to an NFA simulation.

Match many strings against one pattern fast?

Compile the pattern once into a DFA; then each string is scanned in O(len) with no per-string DP. This is the production strategy (RE2).
